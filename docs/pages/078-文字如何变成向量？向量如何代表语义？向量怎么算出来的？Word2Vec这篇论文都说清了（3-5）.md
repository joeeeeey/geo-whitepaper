第四部分：结果 (4 Results)

这部分开始进入实验和验证环节。作者首先讨论了如何科学地评测词向量的质量。

• 传统的评测方法 : 找一个词，比如 “France”（法国），然后看看和它向量距离最近的词是不是 “Italy”（意大利）、“Germany”（德国）这些国家。这种方法很直观，但不够系统和量化。

• 本文提出的更复杂的评测方法 :

• 问题：“What is the word that is similar to small in the same sense as biggest is similar to big ?” (哪个词与 small 的关系，和 biggest 与 big 的关系一样？)

• 解题方法 : 计算 vector("biggest") - vector("big") + vector("small") 。

• 答案 : 在整个词向量空间中，寻找与计算结果向量最接近的词。如果这个词恰好是 “smallest”，那么这个问题就答对了。

• 作者沿用了第二页提到的“类比关系”思想，并将其系统化。

• 他们设计了一个包含成千上万个“类比问题”的测试集。例如：

• 这个评测方法的好处 :

• 它不再是模糊地看“谁和谁近”，而是精确地检验词向量是否捕捉到了特定的 语法关系 （如比较级、最高级）和 语义关系 （如国家-首都，男人-女人）。

• 这是一个可以量化的、打分的目标，非常适合用来比较不同模型的好坏。

• 一个更惊人的例子 :

• France is to Paris as Germany is to Berlin (法国之于巴黎，犹如德国之于柏林)。

• 计算: vector("Paris") - vector("France") + vector("Germany") ，结果最接近的词向量应该是 vector("Berlin") 。

• 这表明训练好的词向量甚至学到了“世界地理知识”！

第五页总结

• 直观理解 : 通过图1，我们清晰地看到了CBOW（多对一）和Skip-gram（一对多）在结构上的根本区别。

• 科学评测 : 作者不再满足于主观的相似词列表，而是设计了一个大规模的“类比问题”测试集，将词向量的质量评估变成了一个可以客观打分的任务。

• 深远影响 : 这种“向量运算 ≈ 语义关系”的思想，以及配套的评测方法，对后来的NLP研究产生了极其深远的影响。它为衡量词向量的“智能程度”提供了一个标尺。

表格1：测试集问题类型举例 (Table 1)

这张表格展示了他们构建的“语义-语法词关系测试集”（Semantic-Syntactic Word Relationship test set）中的部分例子。这个测试集被分成了两大类：

• 语义关系 (Semantic questions) : 考察的是词语的真实世界意义。

1. Common capital city (首都-国家) : Athens : Greece :: Oslo : Norway

2. All capital cities (所有首都-国家) : Astana : Kazakhstan :: Harare : Zimbabwe

3. Currency (货币-国家) : Angola : kwanza :: Iran : rial

4. City-in-state (城市-州) : Chicago : Illinois :: Stockton : California

5. Man-Woman (男性-女性) : brother : sister :: grandson : granddaughter

• 语法关系 (Syntactic questions) : 考察的是词形变化和语法结构。

1. Adjective to adverb (形容词-副词) : apparent : apparently :: rapid : rapidly

2. Opposite (反义词) : possibly : impossibly :: ethical : unethical

3. Comparative (比较级) : great : greater :: tough : tougher

4. Superlative (最高级) : easy : easiest :: lucky : luckiest

5. Present Participle (现在分词) : think : thinking :: read : reading

6. Nationality adjective (国籍形容词) : Switzerland : Swiss :: Cambodia : Cambodian

7. Past tense (过去式) : walking : walked :: swimming : swam

8. Plural nouns (名词复数) : mouse : mice :: dollar : dollars

9. Plural verbs (动词复数) : work : works :: speak : speaks

这个测试集的意义 : 它非常全面，覆盖了从常识到语法的各种关系。一个好的词向量模型应该在这些问题上都取得高分。总共有 8869 个语义问题和 10675 个语法问题。

4.2 准确率最大化 (Maximization of Accuracy)

这部分，作者开始探索如何让模型的准确率（在上述测试集上的得分）变得更高。他们研究了两个关键因素： 词向量的维度（Dimensionality） 和 训练数据的量（Training words） 。

表格2：CBOW模型在不同配置下的表现 (Table 2)

![image]()

这张表格非常重要，它揭示了一个深刻的规律。

• 实验设置 :

• 模型: CBOW

• 训练数据: Google News (新闻语料)，总共约60亿单词。这里用了其中的一小部分（从24M到783M）。

• 词汇表: 限制在前3万个最常见的词。

• 评测: 在上述测试集中，只看那些问题和答案都在这3万词汇表内的问题。

• 数据解读 :

• 类比 : 想象你在画一幅画。 数据量 就像你拥有的 颜料种类 ， 维度 就像你的 画布大小和画笔精细度 。只有更多的颜料配上更大更精细的画布，你才能画出更丰富、更细腻的杰作。只增加颜料，画布太小施展不开；只用大画布，颜料太少则画面单调。

• 横向看 (固定维度，增加数据量) : 比如看 Dimensionality = 300 这一行，随着训练数据从24M增加到783M，准确率从 23.2% 一路提升到 45.9% 。 结论：数据量越大，效果越好。

• 纵向看 (固定数据量，增加维度) : 比如看 Training words = 783M 这一列，随着维度从50增加到600，准确率从 23.2% 提升到 50.4% 。 结论：维度越高，效果越好。

• 关键洞察 (看对角线和整体趋势) : 作者指出，单纯增加数据量或单纯增加维度，带来的收益会逐渐减少（ diminishing improvements ，即边际效益递减）。要想获得最佳效果， 必须同时增加词向量的维度和训练数据的量 。

第六页总结

• 评测标准 : 我们现在有了一个非常全面、包含14种类型的“语义-语法”类比问题测试集，作为评判词向量质量的“标准考试”。

• 成功秘诀 : 通过初步实验（Table 2），我们发现，想要训练出最牛的词向量，不能只靠“堆数据”或“加维度”中的任何一个，而是需要 “双管齐下” ，协同增加数据量和向量维度。

• 引出问题 : 虽然我们知道了“成功秘诀”，但我们还没比较不同模型架构（CBOW, Skip-gram, NNLM等）在同等条件下的表现。哪个模型本身更优秀呢？这正是下一页要回答的问题。
